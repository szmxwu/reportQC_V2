## 数据情况

| 项目 | 详情 |
|------|------|
| 数据文件 | 4个Excel文件 (726MB) |
| 预估记录数 | ~360万条 |
| 字段 | 影像号, 性别, 年龄, 部位, 描述, 结论, 标准化部位等 |
| 主要文本字段 | 描述 + 结论 |

## N-gram模型评估

### 用户反馈的问题
**假阳性高** - 这是N-gram在医学文本中的典型问题：

| 原因 | 说明 |
|------|------|
| 医学术语罕见 | "钙化影"、"条索影"等专业术语在正常文本中频率低 |
| 组合爆炸 | 部位+病变+位置的组合极多，大部分N-gram都是"罕见"的 |
| 缺失上下文 | N-gram只看局部窗口，不理解医学语义 |
| 缺乏负样本 | 真实语法错误太少，无法区分"罕见术语"vs"真实错误" |

### 替代方案对比

| 方案 | 优点 | 缺点 | 适用性 |
|------|------|------|--------|
| **N-gram** | 简单、快速 | 假阳性高 | ⭐⭐ |
| **规则+词典** | 精准、可解释 | 需要人工维护 | ⭐⭐⭐⭐ |
| **Confusion Set** | 针对性强 | 覆盖有限 | ⭐⭐⭐ |
| **Language Model (BERT)** | 理解上下文 | 计算量大、需要GPU | ⭐⭐⭐ |
| **对比学习** | 无需负样本 | 实现复杂 | ⭐⭐ |

### 推荐方案：混合策略

```
第一层: 规则检测 (快速、高精度)
├── 常见错别字库 (typo_database.py) ✅
└── 医学术语白名单 (user_dic_expand.txt) ✅

第二层: 统计过滤 (减少假阳性)
├── N-gram BUT 仅用于"已知错误模式"
├── 基于jieba分词 + 用户词典
└── 只标记与混淆集匹配的低频n-gram

第三层: LLM验证 (可选)
└── 仅对前两层的可疑结果进行验证
```

## 训练步骤

### 阶段1: 数据预处理

```python
# grammer/prepare_training_data.py
"""
1. 从Excel读取数据（分批，避免内存溢出）
2. 合并"描述"+"结论"字段
3. 使用jieba分词（加载user_dic_expand.txt）
4. 生成词频统计
5. 输出: 词汇表 + 共现矩阵
"""
```

**关键优化**：
- 分块读取：每次处理10万条
- 使用jieba.load_userdict()加载专业词典
- 只统计有意义的内容词（过滤停用词）

### 阶段2: 构建Confusion Set增强模型

```python
# grammer/confusion_model.py
"""
基于混淆集(Confusion Set)的检测：

1. 常见混淆对:
   - 文/纹 (肺文里->肺纹理)
   - 渡/度 (低密渡->低密度)
   - 末/未 (末见->未见)
   - 曾/增 (曾强->增强)

2. 检测逻辑:
   if 检测到混淆字 in 文本:
      ngram = 提取上下文 (前后2个字)
      if ngram not in 白名单:
          标记为可疑错误

3. 白名单构建:
   - 从360万条报告中提取所有包含"文"的3-gram
   - 频率>10的加入白名单（如"肺纹理"）
   - 频率<10的标记为待验证
"""
```

### 阶段3: 训练统计模型

```python
# grammer/train_confusion_model.py

# 输入: 360万条医学报告
# 输出:
#   - confusion_whitelist.pkl: 每个混淆字的白名单n-gram
#   - word_frequency.pkl: 词频表
#   - phrase_patterns.pkl: 高频短语模式

# 示例输出结构:
{
    "文": {
        "whitelist": {"肺纹理": 50000, "支气管": 45000, ...},  # 正常
        "suspicious": {"肺文里": 3, "肺文增": 2, ...}          # 可疑
    },
    "渡": {
        "whitelist": {},  # "渡"几乎不应出现在医学报告中
        "suspicious": {"低密渡": 5, "高密渡": 3, ...}
    }
}
```

## 关键问题：解决假阳性

### 策略1: 白名单机制
```python
# 如果一个n-gram在训练数据中出现频率 > threshold，视为正常
WHITELIST_THRESHOLD = 5  # 出现5次以上视为正常
```

### 策略2: 混淆集约束
```python
# 只检测已知混淆字，不检测所有罕见n-gram
CONFUSION_CHARS = ['文', '渡', '末', '曾', '象', '也', '追', ...]

# 原始N-gram的问题：检测到"钙化影"频率低 -> 误报
# 改进方案："钙化"不在混淆集中 -> 跳过
```

### 策略3: jieba分词验证
```python
import jieba
jieba.load_userdict("config/user_dic_expand.txt")

# 分词后检测：理解词边界
# "肺文里增粗" -> ["肺", "文里", "增粗"] -> "文里"不在词典 -> 可疑
# "肺纹理增粗" -> ["肺", "纹理", "增粗"] -> 都在词典 -> 正常
```
